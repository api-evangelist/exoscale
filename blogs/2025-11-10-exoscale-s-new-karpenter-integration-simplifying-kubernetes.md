Exoscale Scalable Kubernetes Service (SKS) has long time support for cluster-autoscaler to automatically adjust the number of nodes in a Kubernetes cluster based on the resource requests of the workloads running in the cluster. This integration has been a key feature for our users, allowing them to optimize their resource usage and reduce costs. This implementation relies on SKS Nodepool implementation.
